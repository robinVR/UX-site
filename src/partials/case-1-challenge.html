The Challenge

Engineers who build products and technologies need access to a high end user experience that makes browsing and reading ARM technical documentation a simple and easy task.

Must be designed for the unknown

There are over 20,000 docs pages with a wide variety of HTML tags, line lengths, images, and code examples.

Load quickly

Documentation is plain text so users expect a fast experience.

Support multiple version of the same document

User must be able to change version of a document.

SEO

Must be able to beat other user generated content ranking on a google search.

Support 4 levels of hierarchy

Users must be able to browse and find books, chapters,
topics and sub-topics.

Support a wide range of devices

Must be readable on mobiles and tablets
